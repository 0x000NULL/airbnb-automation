## ğŸ“± User Flows

### **Flow 1: First-Time Host Setup**
1. Host signs up (email or OAuth)
2. Host adds property (name, location, bedrooms, Airbnb listing ID)
3. Host connects Airbnb account (OAuth or API key)
4. System fetches upcoming bookings
5. Host configures automation (enable/disable for each task type)
6. Host sets budget & preferences
7. System is live!

### **Flow 2: Automatic Turnover Booking**
1. Airbnb booking created by guest
2. System fetches booking via Airbnb API
3. System auto-generates tasks:
   - Cleaning (scheduled 2 hours before checkout)
   - Guest communication (send pre-arrival message)
   - Restocking (if needed)
4. System searches RentAHuman for available cleaners in property location
5. System books cheapest/best-rated human within budget
6. Host notified (email: "Cleaner booked for Property A")
7. Human gets task details + property address
8. Cleaner completes task, uploads photos
9. Host reviews completion, rates human
10. Payment processed via RentAHuman

### **Flow 3: Guest Issue Resolution**
1. Guest messages host on Airbnb ("AC is broken")
2. System detects message via polling
3. System auto-generates maintenance task
4. System searches RentAHuman for "HVAC technician" or "handyman"
5. System books nearest technician
6. Host notified + guest notified
7. Technician completes work, uploads photos/invoice
8. System flags for follow-up ("Was issue resolved?")
